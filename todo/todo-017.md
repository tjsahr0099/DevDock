너는 지금 이 저장소에서 “에이전트 기반 개발 파이프라인”을 자동 구성해야 한다.

목표: 다음 5개 역할을 수행하는 Claude Code 에이전트(또는 스킬/에이전트 조합)를 이 저장소에 생성하고, 실제로 작동 가능하도록 파일/폴더/규칙을 세팅해라.

[역할]
1) TodoWriter: 사용자의 요청을 /todo 폴더에 *.todo.md 파일로 생성. todo는 요청을 간략 요약만 하고 체크리스트를 만든다.
2) Designer: todo를 입력으로 받아 /todo 폴더에 *.design.md 기능 설계 파일 생성(개발 가능한 수준).
3) Implementer: 사용자의 “진행” 요청이 있을 때 다음 미완료 todo를 선택해 구현하고, 완료한 todo를 체크한다(기본은 1개 항목만 처리).
4) Tester: 구현 완료 후 테스트 케이스를 설계하고, 가능한 범위에서 실제 테스트를 실행하며 결과를 기록한다.
5) DocUpdater: 모든 작업이 끝나면 /docs에 기능 변경사항/사용법/테스트결과를 업데이트한다.

[필수 요구사항]
- 이 저장소에 맞는 “Claude Code 에이전트 정의 파일”을 생성해라. (예: .claude/agents/ 아래)
- /todo, /docs/features 등 필요한 폴더를 없으면 생성해라.
- 각 에이전트는 입력/출력 파일 경로 규칙을 명시해야 한다.
- 작업 ID 규칙을 만들어라: /todo 폴더에서 가장 큰 번호 +1로 0001부터 증가하는 방식, 파일명은 "<ID>-<slug>.*" 형태.
- 산출물 포맷을 고정해라(파이프라인이 깨지지 않도록). todo/design/test/docs 각각의 템플릿을 포함해라.
- Orchestrator(메인) 에이전트를 반드시 두고, 사용자의 요청 문장을 보고 어떤 에이전트를 어떤 순서로 호출할지 결정하도록 해라.
- 구현 단계는 안전을 위해 기본적으로 “다음 todo 1개만 처리” 전략을 사용하되, 사용자가 원하면 여러 개를 처리할 수 있게 옵션을 둬라.
- 테스트는 “설계된 테스트 케이스 문서”를 남기고, 가능하면 실제 테스트 커맨드(예: gradle test, mvn test, npm test 등)를 탐색해 실행해라. 실행이 불가하면 실행 불가 사유와 수동 테스트 절차를 기록해라.
- 마지막에는 “설치/사용 방법”을 docs 또는 README에 추가해라. (어떤 명령/어떻게 요청하면 파이프라인이 돌아가는지)

[실행]
- 지금 저장소의 구조를 확인하고(언어/빌드툴/테스트툴 추정), 그에 맞춰 위 에이전트 파일들을 생성하고 커밋 가능한 상태로 만들어라.
- 완료되면 생성/수정된 파일 리스트를 요약하고, 예시 사용 시나리오(요청 문장 예시 5개)를 제공해라.

중요: 너(Claude)는 “최소한의 질문만” 하고, 가능한 한 스스로 판단해서 진행한다.
