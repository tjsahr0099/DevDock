# 호출 흐름 분석기

Spring Boot 기반 Java 프로젝트의 소스 코드를 분석하여 API 엔드포인트와 배치 스케줄러의 호출 흐름을 시퀀스 다이어그램으로 시각화합니다.

## 사용 방법

### 1. 프로젝트 선택
- **찾아보기** 버튼으로 Java 프로젝트의 루트 디렉토리를 선택합니다.
- `src/main/java` 하위의 `.java` 파일들이 분석 대상입니다.

### 2. 프로젝트 분석
- **분석** 버튼을 클릭하면 프로젝트 구조를 파싱합니다.
- 분석이 완료되면 좌측 트리에 결과가 표시됩니다.

### 3. 분석 결과 트리
- **API Endpoints**: Controller의 HTTP 매핑 메서드 목록
  - GET / POST / PUT / DELETE 등 HTTP 메서드별 아이콘 표시
- **Batch/Scheduler**: `@Scheduled` 어노테이션이 붙은 메서드 목록
- 각 항목의 클래스와 메서드가 트리 구조로 표시됩니다.

### 4. 시퀀스 다이어그램
- 트리에서 메서드를 클릭하면 해당 메서드의 호출 흐름이 시퀀스 다이어그램으로 생성됩니다.
- Controller → Service → Repository 계층 간의 호출 관계를 시각화합니다.
- **확대/축소**: `+` / `-` 버튼 또는 `Ctrl + 마우스 휠`

### 5. PlantUML 소스
- **PlantUML 코드** 섹션을 펼치면 생성된 PUML 소스를 확인할 수 있습니다.
- **복사** 버튼으로 클립보드에 복사할 수 있습니다.
- **PUML 저장** 버튼으로 파일로 저장할 수 있습니다.

## 분석 대상
- `@Controller`, `@RestController` 클래스의 API 엔드포인트
- `@Service`, `@Component` 클래스의 비즈니스 로직
- `@Repository`, `@Mapper` 클래스의 데이터 접근 계층
- `@Scheduled` 배치/스케줄러 메서드
- `@Autowired`, `@Inject` 필드 주입 관계

## 참고
- 프로젝트 경로 설정은 자동으로 저장되어 다음 실행 시 복원됩니다.
- 테스트 디렉토리(`src/test`)는 분석에서 제외됩니다.
